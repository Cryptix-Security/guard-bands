# v0.5.0-poc

Two-channel architecture release for Guard Bands.

Classic boundary failures were solved by moving control out of band —
telephony signaling left the voice channel, prepared statements separated SQL
code from data. This release implements that design for LLM applications at
the layer the application controls: untrusted content and trusted
instructions travel through **two separate services, deployable on different
ports, hosts, or network segments**, joined at a single cryptographic
verification point.

## Two-channel reference architecture (`dual_channel/`)

- **Data plane** (`dual_channel/data_plane.py`, e.g. port 8001): intake only.
  Wraps untrusted content into signed inert blocks bound to `channel: data`
  with the data-plane key id and issuer — all authenticated by the MAC. It
  exposes no tool registry, no instruction surface, and no model access, and
  rejects guard-band-marker smuggling at ingest.
- **Control plane** (`dual_channel/control_plane.py`, e.g. port 8002): the
  only service with tools. Admits data solely when the signature proves it
  came through the data plane; rejects tampered, unwrapped, foreign-issuer,
  wrong-channel, and cross-tenant blocks fail-closed. Action selection comes
  exclusively from the authenticated control channel — document text is never
  parsed for instructions, so an injected "call issue_refund" selects
  nothing, and the same escalation through the real channel still faces role
  authorization.
- **Demo**: `make dual-channel-demo` drives both services end to end with no
  LLM API key, showing every rejection path.
- **Tests**: 11 new invariant tests, including forged-issuer and
  wrong-channel bindings. Suite: 72 passing.
- **Docs**: `docs/DUAL_CHANNEL.md` covers topology, enforced invariants, and
  honest limits — the final model hop still converges into one token stream;
  pairing with a quarantined-model (dual-LLM) pattern is the named next step,
  with the Guard Band signature as the capability token between models.

## Security status at release

- Local pytest suite passes: 72 tests.
- GitHub Actions CI and CodeQL passed on the feature commit.
- Branch protection on `main`: required status checks (CI + CodeQL), no
  force pushes or deletions.

This remains a proof-of-concept release. The two-channel architecture is a
reference implementation intended for evaluation, extension, and bypass
attempts.
