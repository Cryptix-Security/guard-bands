# v0.2.0-poc

Pilot-readiness release for Guard Bands.

This release moves the project beyond the initial cryptographic boundary POC by adding practical controls that a small team would need before evaluating Guard Bands in an internal application.

Notable additions:

- SQLite-backed persistent replay ledger for single-node pilots.
- FastAPI verification middleware for protected request-body routes.
- Authorization helper examples for role, tenant, user, action, and policy-path checks.
- Reference support-ticket app showing wrap, verify, authorize, allow, and reject flows.
- API-key-free local demos: `make demo` and `make reference-demo`.
- Parser and verification micro-benchmarks via `make bench`.
- Production deployment guide with TLS, SSO, replay ledger, audit, and key-management expectations.
- Architecture, limits, authorization, and reference-app documentation.
- Expanded tests for FastAPI middleware, replay persistence, authorization, and sensitive tool-call enforcement.

Security status at release:

- GitHub Actions CI passes on Python 3.11 and 3.12.
- CodeQL passes.
- Open CodeQL alerts: none.
- Open Dependabot alerts: none.

This is still a proof-of-concept release. The SQLite replay ledger is suitable for a single-node pilot, not a multi-replica production deployment. Production use still requires environment-specific key management, authorization design, shared replay storage, TLS, audit retention, monitoring, and operational review.
