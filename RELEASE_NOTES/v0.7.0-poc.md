# v0.7.0-poc

Python SDK release for Guard Bands.

This release adds a first-party Python SDK that makes the main Guard Bands API
and the two-channel reference architecture easier to integrate into existing
applications.

## Python SDK (`guardbands_sdk`)

- `GuardBandsClient` for the main FastAPI service:
  - `/wrap`
  - `/verify`
  - `/chat/estimate-cost`
  - `/chat`
  - `/health`
- `DataPlaneClient` for `dual_channel.data_plane`:
  - `/ingest`
  - `/health`
- `ControlPlaneClient` for `dual_channel.control_plane`:
  - `/execute`
  - `/health`
- Typed dataclass responses for wrap, verify, chat, cost estimates, ingest,
  wrapped documents, and control-plane execution.
- Domain exceptions for common API outcomes:
  - `VerificationFailed`
  - `CostThresholdExceeded`
  - `AuthenticationError`
  - `AuthorizationError`
  - `RateLimitError`
  - `NotFoundError`
  - `GuardBandsAPIError`

## Examples and docs

- Added `docs/PYTHON_SDK.md` with install, main API usage, two-channel usage,
  and exception handling.
- Added `examples/sdk_quickstart.py` for the main API.
- Added `examples/sdk_dual_channel.py` for the data-plane/control-plane flow.

## Packaging

- Added PEP 517 build metadata.
- Added package discovery so `guardbands_sdk` is included in built wheels.
- Built and smoke-tested `guard_bands-0.7.0-py3-none-any.whl`.

## Verification

- Local pytest suite passes: 94 tests.
- SDK test coverage added with `httpx.MockTransport`.
- `compileall` and `git diff --check` pass.

The SDK is intentionally thin. It improves integration ergonomics but does not
become the security boundary: verification, replay protection, authorization,
and tool-call enforcement remain server-side responsibilities.
