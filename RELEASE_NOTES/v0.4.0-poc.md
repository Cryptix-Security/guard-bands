# v0.4.0-poc

Production-readiness release for Guard Bands.

This release closes the Tier 1/Tier 2 gaps that stood between the POC and a
small-company pilot: how secrets are supplied, how the stack is fronted with
TLS, and how the container runs. The cryptographic boundary was already
production-shaped; this release hardens the deployment surface around it.

## Secrets management

- Pluggable secret provider for secret-bearing settings (`SECRET_KEY`,
  `GUARD_BAND_KEYS`, `ANTHROPIC_API_KEY`, audit tokens/DSNs):
  - `SECRETS_BACKEND=env` (default) — also covers orchestrator-injected secrets
    (Vault Agent, ECS task secrets, Kubernetes External Secrets)
  - `SECRETS_BACKEND=aws` — AWS Secrets Manager
  - `SECRETS_BACKEND=vault` — HashiCorp Vault (KV v2)
- Cloud SDKs are optional extras (`pip install 'guard-bands[aws]'` / `[vault]`)
  so the base image stays light. A missing secret falls back to a default; a
  backend/credential error fails loudly (fail closed).
- See [`docs/SECRETS.md`](../docs/SECRETS.md).

## Deployment hardening

- Production Compose overlay (`deploy/docker-compose.prod.yml`):
  - Caddy TLS front door (self-signed by default; automatic Let's Encrypt for a
    real domain)
  - no service publishes a host port except the proxy
  - Keycloak runs in production mode (`start`, not `start-dev`)
  - restart policies and CPU/memory limits on every service
- The container image now runs as a non-root user with a `/health` healthcheck.
- Added `.env.production.example` with hardened defaults (replay protection on,
  SSO enforced, `DEBUG=false`).

## Not in scope (still operator responsibility)

- Application-specific authorization for your sensitive tools (the pattern
  ships in `reference_app/` and `docs/AUTHORIZATION.md`; the policy is yours).
- A shared/distributed replay ledger for multi-replica deployments (single-node
  SQLite is supported today).
- Audit retention, alerting, and backup/restore testing — operational, per
  environment. See [`docs/PRODUCTION_DEPLOYMENT.md`](../docs/PRODUCTION_DEPLOYMENT.md).

## Security status at release

- Local pytest suite passes (60 tests).
- CI and CodeQL run on push.

This remains a proof-of-concept. It is now shaped for a single-node small-company
pilot; multi-replica scale, HSM/KMS key custody, and a policy engine remain
future work.
